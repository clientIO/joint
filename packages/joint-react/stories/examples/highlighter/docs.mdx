# Using Highlighters

This example demonstrates how to use JointJS highlighters to create visual emphasis on elements in your diagram.

## Overview

Highlighters are useful for:
- **Hover effects**: Show visual feedback when users interact with elements
- **Selection states**: Indicate which elements are currently selected
- **Focus indicators**: Highlight elements that have focus or are being edited
- **Status indicators**: Show different states (active, disabled, error, etc.)

## Available Highlighter Types

### Mask
Creates a border/mask around the element.

```tsx
highlighters.mask.add(elementView, 'root', 'my-highlighter', {
  padding: 5,
  attrs: { stroke: 'blue', 'stroke-width': 2 },
});

highlighters.mask.remove(elementView, 'my-highlighter');
```

### Opacity
Changes the opacity of the element.

```tsx
highlighters.opacity.add(elementView, 'root', 'my-highlighter', {
  alphaValue: 0.5,
});

highlighters.opacity.remove(elementView, 'my-highlighter');
```

### Stroke
Adds a stroke outline around the element.

```tsx
highlighters.stroke.add(elementView, 'root', 'my-highlighter', {
  padding: 10,
  rx: 5,
  ry: 5,
  attrs: { stroke: '#333', strokeWidth: 3 },
});

highlighters.stroke.remove(elementView, 'my-highlighter');
```

## Best Practices

1. **Use paper events**: Subscribe to `element:mouseenter` / `element:mouseleave` via `usePaperEvents` for hover effects
2. **Use state + useEffect**: For selection-driven highlighting, store selected ID in state and manage highlighters in a `useEffect`
3. **Unique IDs**: Use a consistent highlighter ID (e.g. `'hover'`, `'selection'`) so adding a new one replaces the previous
4. **Cleanup**: Use `highlighters.*.remove(elementView, id)` or `highlighters.*.removeAll(paper, id)` for cleanup
