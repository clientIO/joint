curve.tAtLength(length [, opt])

Return the t value of the point that lies length away from the beginning of the curve.

If the length requested does not lie on the curve (either because it is smaller than 0 or because it is higher than curve length), 0 or 1 are returned instead, respectively.

The curve is first subdivided, according to opt.precision + 1 (refer to curve.length() documentation for more information about precision and curve flattening). Then, one subdivision is identified which contains the point at length. A binary search is then performed on that subdivision, until a curve is found whose endpoint lies within opt.precision + 1 away from length. The t value of that endpoint is returned by the function (refer to curve.pointAtLength() documentation for more information).

The default value for opt.precision is 2; this corresponds to maximum observed error of 1%.

The opt.subdivisions property may be specified, directly providing an array of pre-computed curve subdivisions from which to calculate curve length. Use the curve.getSubdivisions() function to obtain an array of curve subdivisions.

The opt.precision property is still necessary, however; it determines the precision of the point search algorithm. Beware that opt.precision will not match the precision of opt.subdivisions unless the subdivisions were created with precision of opt.precision + 1!
