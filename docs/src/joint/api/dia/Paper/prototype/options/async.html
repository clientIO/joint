async - when true, the paper uses asynchronous rendering to display cells added to the graph (either with graph.resetCells() or graph.addCells() methods). This is very useful for adding a large number of cells into the graph. The rendering performance boost is significant and it doesn't block the UI. When asynchronous rendering is used, some of the cell views might not yet be present the paper when one tries to access them (with paper.findViewByModel(), element.findView() or link.findView() methods). You should only use those access methods after the paper lets you know it is safe - with the 'render:done' event.
