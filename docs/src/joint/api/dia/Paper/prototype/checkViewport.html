paper.checkViewport([viewportFn])

For every view in the paper, determine if it fits into a specified viewport; if it does not fit, remove the view from the DOM.

While async papers do this automatically (if the viewport option is provided), synchronous papers require a manual call to this function for this functionality to be applied. To reverse the effect of this function, use paper.dumpViews().

If viewportFn is provided, use it as the callback function; otherwise, use the function specified in paper.options.viewport. In any case, the callback function is provided with three arguments:

view mvc.View The view in question
isDetached boolean Is the view currently hidden from the paper?
paper dia.Paper This paper (for context)

See examples of usage in the documentation of the viewport option.
