Multiple papers

More papers for the same graph

Now when you learned that a joint.dia.Paper is a view for the joint.dia.Graph model, you can imagine that the same model can have multiple views. That's right. Let's use the same application as the Hello World! above and let's add another paper, scaled and non-interactive, that serves as a small overview of the graph. 

var paperSmall = new joint.dia.Paper({
    el: document.getElementById('myholder-small'),
    width: 600,
    height: 100,
    model: graph
});
paperSmall.scale(.5);
paperSmall.$el.css('pointer-events', 'none');

Try to interact with the first, bigger, paper. You should see all the changes reflected in the other, smaller, paper.

More views for the same element

Similary, we can use a different view for the same element/link model. i.e. joint.dia.ElementView is a view for the joint.dia.Element. Here's an example how to write a simple element view that always renders a rectangle ignoring all the presentation attributes stored on the model.

var paper = new joint.dia.Paper({
    el: document.getElementById('myholder'),
    width: 600,
    height: 100,
    model: graph,
    elementView: joint.dia.ElementView.extend({
        renderMarkup: function() {
            this.velBody = V('rect', {
                fill: 'lightgray',
                stroke: 'gray'
            }).appendTo(this.vel.empty());
        },
        resize: function() {
            this.velBody.attr(this.model.size());
        },
        update: function() {
            /* Noop: we simply ignore all attrs attributes. */
        }
    });
});

Please see the source code or Creating Custom Views section for more informaiton.
